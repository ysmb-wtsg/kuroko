# ADR 0009: 外部ファイラの差し替え（krk.opt.file_manager）

## ステータス

採用（2026-06-15）

## コンテキスト

サイドパネルのファイルツリーは内蔵 `FileTreePane` 固定だった。一方で yazi / lf など
高機能な外部TUIファイラを使い慣れたユーザーは、それを kuroko のサイドパネルとして
使いたいという要求がある。

kuroko には既に外部TUIをPTYで起動する基盤（`TerminalPane::with_command()`）があり、
Gitパネル（`krk.opt.git_tool` で lazygit/tig）とエディタ（`krk.opt.editor` で vim/nvim）が
同じパターンで差し替え可能になっている。ファイラも同じ仕組みに乗せられる。

検討した代替案:

- **内蔵ファイラと外部ファイラを別ペインとして併存させる**: 連携機能を維持しつつ外部
  ファイラも使える利点があるが、新しいトグルキー・ペインID管理・レイアウト分岐が増え、
  「グローバルな状態を増やさない」方針に反する。YAGNI。今回は見送る
- **内蔵ファイラに yazi 風の機能を作り込む**: 再実装コストが高く、使い慣れた外部ツールを
  そのまま使いたい要求にも応えられない

## 決定

1. **`krk.opt.file_manager` を追加する**。未設定または `"builtin"` のときは内蔵 `FileTreePane`、
   それ以外は指定された外部ファイラを起動する**完全置き換え**方式とする
2. **外部ファイラは既存のPTY基盤（`TerminalPane::with_command()`）で起動する**。
   git_tool / editor と同じパターン。コマンド文字列は `parse_command()` で
   プログラム名と引数に分割する（editor と共用、例 `"yazi --opt"`）
3. **コマンドが見つからない場合は警告を出して内蔵ファイラにフォールバックする**。
   git_tool（未検出時は表示しない）と異なり、ファイラはサイドパネルとして常に何かを
   表示すべきなので、内蔵ツリーへフォールバックする
4. **生成したペインは従来どおり `file_tree_id` に格納する**。トグル・フォーカス・リサイズは
   既存ロジックがそのまま動く。`TerminalPane` は `wants_raw_input()=true` を返すため、
   直通状態の入力はファイラに流れる（ADR 0005 / 0007 と整合）

## 結果

- ユーザーは慣れた外部ファイラをサイドパネルで使える
- git_tool / editor と一貫した設定体験（`krk.opt.*` で外部ツールを指定）
- トレードオフ:
  - 外部ファイラ採用時、内蔵ファイラのkuroko連携キー（`o` エージェント送信 / `e` 内蔵
    エディタ / `p` プレビュー / `a`,`d`,`r` 作成・削除・リネーム）は無効になり、ファイラ側の
    キー操作に委ねられる。これは「完全置き換え」を選んだ結果で、状態を増やさない方針を優先した
  - 外部ファイラからエージェントへファイルを渡す等の連携は将来課題（未実装）
