# ADR 0003: ボーダーレスペインとセパレータ線によるペイン境界表現

## ステータス

採用（2026-06-13）

## コンテキスト

従来、全ペインが `Borders::ALL` で全周ボーダーを描画していた。これには以下の問題があった。

1. **表示領域の浪費**: 隣接ペイン間で線が二重になり、1ペインあたり上下左右4セルを消費していた
2. **PTYリサイズの発生**: AgentPaneはフォーカス時のみボーダーを描画していたため、フォーカスの出入りで内容領域が±2セル変化し、毎回PTYリサイズ→ターミナル出力の再フローが発生していた
3. **情報の分散**: ペインタイトル・エージェント状態バッジ・モード表示がペイン枠とステータスバーに分散し、視線の流れが定まらなかった

## 決定

1. **全ペインのボーダーを廃止する**。`Pane::render` は渡された領域全体に内容を描画し、フォーカス状態で描画領域を変えない
2. **ペイン境界はレイアウト分割面の1セル幅セパレータ線で表現する**。`LayoutNode::resolve_with_separators` が分割時にセパレータ領域を確保して返し、App側が線を描画する（tmux方式）。分割軸が3セル未満の場合はセパレータを省略する
3. **フォーカス表示はセパレータ色とステータスバーで行う**。フォーカス中ペインに隣接するセパレータは `border_focus` 色、それ以外は `border_subtle` 色で描画する
4. **ペイン枠にあった情報はステータスバー・タブバーに集約する**。モード短縮表示（NOR/INS/SEL）、フォーカス中ペイン名、エージェント状態（タブのステータスドットとステータスバーの集約表示）、レベル付き一時メッセージ（Info/Warn/Error）
5. **ボトムタブバーは `bottom_visible` の間は常に表示する**。タブが2個になった瞬間のレイアウトシフトを防ぐ

## 結果

- 表示領域が広がり、フォーカス切替によるPTYリサイズが消滅する
- フローティングオーバーレイ（プレビュー、パレット、プロンプト、情報）は従来通りボーダーを持つ。タイル領域とモーダルの視覚的区別として意図的に残す
- ペインのモジュール（terminal/filetree/agent）はボーダー描画の責務を持たなくなり、境界描画はApp（render.rs）に一元化される
