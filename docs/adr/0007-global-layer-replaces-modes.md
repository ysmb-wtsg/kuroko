# ADR 0007: モード廃止とグローバルレイヤー導入

## ステータス

採用（2026-06-13）

## コンテキスト

Insert/Normal/Selectのモードモデルには以下の問題があった。

1. **Escの横取り**: InsertモードのEscが常にNormal移行に消費され、PTYに届かない。ターミナルペイン内のvim（Insert脱出）やClaude Code CLI（生成中断）が機能しない
2. **モード意味論の崩壊**: ADR 0005で非rawペインはNormal中もキーが直接流れる特別扱いになり、「Normalモード」の挙動がペイン種別で別物になった。モード表示が実態を説明できない（例: filerフォーカス中はリサイズ`H/J/K/L`が効かない）
3. **表示の有用性**: ユーザーが知りたいのは「いまどのパネルにいるか」であり、モード名ではない

検討した代替案:

- **Ctrl系キーの直接予約**（zellij方式）: 内側アプリのキー（Ctrl+h=BS等）と恒常的に衝突する。zellijがlocked modeを後付けした前例から不採用
- **ワンショットprefix**（tmux方式）: 衝突面は最小だが、リサイズ等の連打操作で毎回prefixを打つコストが高い

## 決定

1. **`Mode` enum（Insert/Normal/Select）を廃止する**。`Action::SetMode` も削除
2. **通常状態（直通）**: Escを含む全キーイベントをフォーカス中ペインへルーティングする。rawペインへはPTYバイト列、非rawペインへは`AppEvent`。アプリはデフォルトでは一切キーを奪わない
3. **グローバルレイヤー**: `Ctrl+g`（デフォルト、変更可）で入る持続状態。単発キーでグローバル操作を行い、`Esc`・`i`・再度`Ctrl+g`で直通に戻る。連打操作（リサイズ等）に対応するためワンショットではなく持続型とする
4. **レイヤー中の挙動はペイン種別に依存しない**。キーは一切ペインへ流れない（ADR 0005の非rawペイン特別扱いを廃止）
   - `h/j/k/l` フォーカス移動 / `H/J/K/L` リサイズ / `Tab`・`S-Tab` 順送り
   - `f/t/g` パネルトグル / `n/x/w/r/]/[/1-9` タブ操作 / `:` パレット / `Enter` コピーモード / `q` 終了
5. **Selectモードはfiler内部状態に移す**。filerフォーカス中（直通）に`v`でトグル。コピーモードと同様「ペイン内部の状態」として扱い、グローバルな概念にしない
6. **ステータスラインはフォーカス中ペイン種別を常時表示**し、レイヤー中は`GLOBAL`バッジを追加表示する
7. **Luaキーマップはコンテキスト制**: `krk.keymap.set("global"|"direct", key, callback)`。`direct`コンテキストへの登録は「直通中でもアプリが先取りするキー」をユーザーが明示的に作る手段（例: `<C-h>`での直接フォーカス移動）。デフォルトは空＝衝突ゼロ。トグルキーは`krk.keymap.set_toggle_key(key)`で変更可能

## 結果

- ターミナルペイン内のvim/Claude CodeでEscが正常に機能する
- filerフォーカス中でもレイヤー経由でリサイズ・パネルトグルが統一的に効く
- 直通中はkurokoがキーを奪わないため、内側アプリとのキー衝突はユーザーが`direct`登録した分だけに限定される
- トレードオフ: グローバル操作に必ず`Ctrl+g`の1打が先行する。頻用操作を1打にしたいユーザーは`direct`コンテキストで自己責任のバインドを作る（README に設定例を記載）
- トグルキーのデフォルト選定の経緯:
  - `Ctrl+Space`: NUL(0x00)が一部端末（WezTerm、macOSのIME割り当て）で届かず実機で機能しなかったため不採用
  - `Ctrl+j`: raw mode中はEnter(CR)と区別でき全端末に届くが、`Ctrl+j`はエージェント（Claude Code等）のマルチライン改行挿入に使われるため、直通中にその用途を奪う。改行はモードレスの主要ユースケースであり不採用
  - `Ctrl+g`: 改行系と無関係・全端末に確実に届く・エージェントが入力に使わない。emacs/readlineのkeyboard-quitとのみ衝突するが影響は限定的。これを採用
- 用途が衝突するユーザーは`set_toggle_key`で変更する
- 直通中の改行: Enterは送信/確定、`Ctrl+j`(LF)は改行挿入とする。kittyキーボードプロトコル（`DISAMBIGUATE_ESCAPE_CODES`）を対応端末で有効化しEsc/Ctrl系の曖昧性を解消するが、DISAMBIGUATEはShift単独修飾を区別しないため`Shift+Enter`は非対応端末で検出できない。よって`Ctrl+j`をポータブルな改行手段として正式化し、`Shift+Enter`/`Alt+Enter`は対応端末でのみLFとして送る。全キーをCSI u化する`REPORT_ALL_KEYS_AS_ESCAPE_CODES`はIME入力を壊すリスクがあるため採用しない
- フォーカス中ペインの端末カーソルを`set_cursor_position`で表示する。直通かつ非オーバーレイ時のみとし、グローバルレイヤー中・コピーモード中・スクロール中・カーソル非表示(DECTCEM)時は出さない。レイヤー中は入力がペインに行かないため非表示にし、入力先を視覚的に区別する
